# Контекст проекта (.claude_context.md)

Краткий ориентир по проекту для Claude. Подробные правила архитектуры —
в `arhitecture.md` (он обязателен к соблюдению, импортируется отдельно).

## Что это за проект

**TasinMobile** — Inertia-монолит на Laravel + Vue 3. Состоит из двух частей:

- **Админка** — управление каталогом (товары, категории и т.д.). Базовая
  часть уже готова (коммит «Tasin mobile admin side is done»).
- **Витрина (storefront / shop)** — публичная клиентская часть каталога.
  В активной разработке. Код в `app/Http/Controllers/Shop/`,
  `resources/js/Pages/Shop/`, `resources/js/Components/Shop/`,
  маршруты в `routes/shop.php` (регистрируются в `bootstrap/app.php`).

## Стек

- PHP 8.2+, Laravel 11
- Inertia.js 2 + Vue 3 (`<script setup>`, Composition API)
- Pinia (глобальный стейт), Tailwind CSS 3, MySQL
- Reverb (WebSockets), Sanctum, Ziggy
- Сборка: Vite 6 (`npm run dev` слушает на `192.168.1.4`; `npm run build`)
- Тесты: Pest / PHPUnit. Статанализ: Larastan. Стиль: Pint.

## Где что лежит

- Backend слои: `app/Http/Controllers`, `app/Services`, `app/Repositories`,
  `app/Models`. Поток: Route → Controller → Service → Repository → Model.
- Frontend: `resources/js/Pages`, `resources/js/Components`,
  `resources/js/Composables`, `resources/js/stores`.
- Маршруты: `routes/web.php` (админка), `routes/shop.php` (витрина).
- `app/Support/` — вспомогательные классы.

## Ключевые правила (детали — в arhitecture.md)

- Вся логика над данными (поиск, фильтрация, сортировка, пагинация) — на
  backend. Frontend только шлёт параметры через Inertia и отображает ответ.
- Eloquent-запросы — только в Repository. Бизнес-логика — только в Service.
- Form Request на каждый POST/PUT/PATCH, `messages()`/`attributes()` на русском.
- Проект одноязычный (русский). Без vue-i18n / `lang/` для UI-текстов.
  (Замечание: в `package.json` ещё висит зависимость `vue-i18n` — наследие,
  в новом коде не использовать.)
- Tailwind only; dark mode через `dark:` классы.
